# Git 代理管理工具

一款简单易用的图形化工具，用于快速管理 Git 代理设置，解决 GitHub 推送失败问题。

## 功能特性

- ✅ **图形界面** - 直观的操作界面，无需记忆命令
- ✅ **一键操作** - 快速设置/取消 Git 代理
- ✅ **实时状态** - 显示当前 HTTP/HTTPS 代理配置
- ✅ **自定义配置** - 支持修改代理 IP 和端口
- ✅ **自动检测** - 启动时自动检测 Git 安装
- ✅ **配置持久化** - 自动保存配置到本地文件

## 系统要求

- **Python**: 3.6 或更高版本
- **Git**: 已安装并添加到系统 PATH
- **操作系统**: Windows / macOS / Linux

## 安装与使用

### 1. 下载文件

下载项目中的 `proxy_gui.py` 文件到本地。

### 2. 启动工具

#### 方式一：双击运行（Windows）
直接双击 `proxy_gui.py` 文件即可启动。

#### 方式二：命令行启动
```bash
python proxy_gui.py
```

### 3. 界面说明

启动后会显示如下界面：

### 4. 操作指南

#### 查询代理状态
点击「**查询状态**」按钮，查看当前 Git 代理配置。

#### 设置代理
1. 在「IP地址」和「端口」输入框中填入代理服务器信息
2. 点击「**保存配置**」按钮保存设置
3. 点击「**设置代理**」按钮应用代理

#### 取消代理
点击「**取消代理**」按钮，清除 Git 代理配置。

#### 修改默认配置
1. 修改「IP地址」和「端口」输入框中的值
2. 点击「**保存配置**」按钮
3. 配置会保存到 `proxy.ini` 文件，下次启动自动加载

## 配置文件

首次运行会自动创建 `proxy.ini` 文件，位于程序同级目录。

默认配置：
```ini
[proxy]
host = 127.0.0.1
port = 7890
```

你也可以手动编辑此文件来修改默认配置。

## 常见问题

### Q: 启动时提示"未检测到 Git"
**A:** 请先安装 Git：
- Windows: 下载安装程序 https://git-scm.com/downloads
- macOS: `brew install git`
- Linux: `sudo apt-get install git` (Ubuntu/Debian)

安装后确保 `git` 命令可以在终端中正常使用。

### Q: 设置代理后仍无法推送到 GitHub
**A:** 请检查以下几点：
1. 确认代理服务正在运行（如 Clash、V2Ray、Shadowsocks 等）
2. 确认代理 IP 和端口填写正确
3. 尝试在浏览器中访问 https://github.com 测试代理是否正常工作
4. 使用 `git config --global http.proxy` 命令验证代理是否设置成功

### Q: 如何恢复默认配置？
**A:** 删除 `proxy.ini` 文件，然后重新启动程序即可恢复默认配置（127.0.0.1:7890）。

### Q: 支持 SOCKS5 代理吗？
**A:** 当前版本只支持 HTTP/HTTPS 代理。如需 SOCKS5 支持，可以手动设置：
```bash
git config --global http.proxy socks5://127.0.0.1:1080
```

## 技术说明

- 使用 **Python tkinter** 构建图形界面，无需安装额外依赖
- 使用 **configparser** 管理配置文件

- 支持 Windows、macOS、Linux 三大平台

## 开发者信息

### 如何修改版本号

版本号定义在 `proxy_gui.py` 文件的顶部：

```python
# ============================================
# 版本号定义 - 修改此处即可更新版本
# 格式：主版本.次版本.修订号
# ============================================
__version__ = "1.0.0"
```

修改 `__version__` 变量的值即可更新软件版本号，版本号会显示在窗口标题中。

### 配置文件说明

- **自动保存**：关闭软件时会自动保存当前配置
- **配置文件位置**：与程序同目录的 `proxy.ini`
- **手动保存**：点击界面上的「保存配置」按钮
